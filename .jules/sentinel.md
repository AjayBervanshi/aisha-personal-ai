## 2026-04-11 - Missing CSRF Validation in OAuth Flow
**Vulnerability:** The `/instagram_setup` command generated an OAuth URL without a `state` parameter, and the callback endpoint `_handle_instagram_oauth_callback` did not validate it. This allowed attackers to perform Cross-Site Request Forgery (CSRF).
**Learning:** When implementing OAuth, a `state` parameter must be passed and securely verified upon the callback. Global in-memory sets or caches for storing this state without tying it to a user act as "security theater" and can be bypassed or cause memory leaks.
**Prevention:** Generate a cryptographically secure `state` parameter using `secrets.token_urlsafe()`, bind it securely to the user's session (e.g., using `time.time()` for expiration and mapping it to `user_id`), and validate it strictly on the callback endpoint before exchanging the authorization code.
